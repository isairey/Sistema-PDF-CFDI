| Latest   | ✅ |
| Stable   | ✅ |
| Legacy   | ⚠️ Limitado |
| Deprecated | ❌ |

---

# 🚨 Reportar vulnerabilidades

Si descubres una vulnerabilidad de seguridad, por favor repórtala de manera responsable.

## 📩 Contacto de seguridad

```bash
security@proyecto.com
```

O mediante:

- GitHub Security Advisories
- Email privado
- Canal oficial del proyecto

---

# ⚠️ Qué NO hacer

Para proteger a los usuarios y al proyecto:

- ❌ No publiques exploits públicamente
- ❌ No compartas credenciales
- ❌ No abras issues públicos con información sensible
- ❌ No ataques servicios de producción

---

# ✅ Qué incluir en el reporte

Por favor proporciona:

- 📋 Descripción clara de la vulnerabilidad
- 🧪 Pasos para reproducir
- 📸 Evidencia o capturas
- 💻 Entorno afectado
- ⚡ Impacto estimado
- 🔧 Posible solución (opcional)

---

# 🔍 Proceso de seguridad

## 📥 1. Recepción

El reporte será recibido y revisado por el equipo de seguridad.

---

## 🧪 2. Validación

Se verificará:

- Reproducibilidad
- Impacto
- Severidad
- Riesgo potencial

---

## 🛠️ 3. Corrección

Se trabajará en:

- Parches de seguridad
- Actualizaciones
- Mitigación temporal

---

## 📢 4. Publicación

Una vez solucionado:

- Se liberará actualización
- Se notificará a la comunidad
- Se documentará el cambio

---

# ⏱️ Tiempo de respuesta

Objetivos aproximados:

| Tipo | Tiempo estimado |
|------|----------------|
| Confirmación inicial | 24 - 72 horas |
| Evaluación | 3 - 7 días |
| Corrección crítica | Prioridad alta |
| Actualización pública | Según complejidad |

---

# 🔒 Buenas prácticas recomendadas

## 👨‍💻 Para desarrolladores

- 🔐 Usar variables de entorno
- 🛡️ Validar entradas
- 📦 Mantener dependencias actualizadas
- ⚡ Aplicar principio de mínimo privilegio
- 🔒 No subir secretos al repositorio

---

## 🖥️ Para despliegues

- 🌐 Usar HTTPS
- 🔑 Configurar autenticación segura
- 🧱 Usar firewalls y protección de red
- 📊 Monitorear logs
- 💾 Mantener backups

---

# 📦 Dependencias y actualizaciones

Se recomienda:

```bash
npm audit
```

o

```bash
pip-audit
```

o

```bash
composer audit
```

para verificar vulnerabilidades en dependencias.
